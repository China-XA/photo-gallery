# 图片相册项目配置指南

## 概述

本项目使用 GitHub 作为图片存储后端，所有用户上传的图片都会保存到 GitHub 仓库中，确保所有设备都能看到相同的内容。

## 前置要求

1. 一个 GitHub 账号
2. 一个 Vercel 账号（用于部署）
3. 一个 GitHub 仓库（用于存储图片和数据）

## 配置步骤

### 第一步：创建 GitHub 仓库

1. 登录 GitHub
2. 点击右上角 "+" → "New repository"
3. 设置仓库名称（例如：`photo-gallery-storage`）
4. 选择 "Public" 或 "Private"（推荐 Private）
5. 勾选 "Add a README file"
6. 点击 "Create repository"

### 第二步：创建必要的文件结构

在你的新仓库中创建以下文件和目录：

1. 创建 `data` 目录
2. 在 `data` 目录下创建 `images.json` 文件，内容如下：
```json
{
  "images": [],
  "categories": []
}
```
3. 创建 `images` 目录（这个目录会自动保存上传的图片）

### 第三步：生成 GitHub Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token" → 选择 "Generate new token (classic)"
3. 设置以下选项：
   - Note: `Photo Gallery Upload`
   - Expiration: 选择一个合适的期限（建议 "No expiration" 或者长期）
   - Scopes: 勾选 `repo`（完整仓库访问权限）
4. 点击 "Generate token"
5. **重要！** 复制生成的 token（只会显示一次）

### 第四步：在 Vercel 中配置环境变量

1. 登录 Vercel
2. 进入你的项目
3. 点击 "Settings" → "Environment Variables"
4. 添加以下三个环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `GITHUB_TOKEN` | 第三步生成的 token | GitHub 访问令牌 |
| `REACT_APP_GITHUB_OWNER` | 你的 GitHub 用户名 | 例如：`your-username` |
| `REACT_APP_GITHUB_REPO` | 仓库名称 | 例如：`photo-gallery-storage` |

**注意：**
- 环境变量名称要完全一致
- `REACT_APP_GITHUB_OWNER` 是你的 GitHub 用户名，不是邮箱
- 确保所有三个变量都已正确添加

### 第五步：重新部署 Vercel 项目

1. 在 Vercel 项目页面，点击 "Deployments"
2. 点击最新部署旁边的三个点
3. 选择 "Redeploy"
4. 等待部署完成

### 第六步：测试上传功能

1. 访问你的网站
2. 登录后尝试上传一张图片
3. 检查你的 GitHub 仓库，应该能看到：
   - `images` 目录下有新上传的图片文件
   - `data/images.json` 文件已更新，包含新图片的信息
4. 在另一台设备上访问网站，应该能看到相同的图片

## 常见问题

### 问题1：上传时显示错误提示

**解决方案：**
1. 检查 Vercel 环境变量是否正确配置
2. 确认 GitHub token 是否有效
3. 检查 Vercel 部署日志获取更多错误信息

### 问题2：图片上传成功但其他设备看不到

**解决方案：**
1. 确认点击了"同步"按钮
2. 检查 GitHub 仓库中的 data/images.json 是否更新
3. 清除浏览器缓存后重新加载

### 问题3：GitHub token 过期了怎么办

**解决方案：**
1. 按照第三步生成新的 token
2. 在 Vercel 环境变量中更新 `GITHUB_TOKEN`
3. 重新部署项目

### 问题4：如何更改 GitHub 仓库

**解决方案：**
1. 在 GitHub 上创建新仓库
2. 在 Vercel 中更新 `REACT_APP_GITHUB_REPO` 环境变量
3. 重新部署

## 本地开发配置（可选）

如果你想在本地开发时也使用 GitHub 存储：

1. 在项目根目录创建 `.env` 文件
2. 添加与 Vercel 相同的环境变量：
```env
REACT_APP_GITHUB_OWNER=your-username
REACT_APP_GITHUB_REPO=your-repo-name
GITHUB_TOKEN=your-token
```
3. 运行 `npm run dev` 启动开发服务器

## 安全建议

1. 不要将 `.env` 文件提交到 Git 仓库
2. 定期更换 GitHub token
3. 如果项目是公开的，建议使用 Private 仓库存储图片
4. 限制 GitHub token 的权限范围

## 验证配置成功的标准

✅ 图片上传成功
✅ GitHub 仓库中能看到新上传的图片
✅ `data/images.json` 文件已更新
✅ 其他设备访问时能看到相同的图片
✅ 点击"同步"按钮能刷新显示
