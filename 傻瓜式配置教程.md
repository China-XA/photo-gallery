# 📸 图片相册 - 傻瓜式配置教程

## 前言

你好！这是一个专门为电脑小白准备的教程，每一步都有详细说明，跟着做就可以了！

## 📋 你已经有的信息

✅ 项目代码仓库：`photo-gallery`
✅ 图片存储仓库：`photo-gallery-storage`

---

## 🎯 第一步：配置图片存储仓库

### 1.1 创建必要的文件结构

1. 打开你的 GitHub，找到仓库 `photo-gallery-storage`
2. 点击仓库页面上的 **"Create new file"**（创建新文件）按钮
3. 在文件名输入框输入：`data/images.json`（注意是 data 文件夹下面的 images.json）
4. 在编辑区域复制粘贴以下内容：

```json
{
  "images": [],
  "categories": []
}
```

5. 向下滚动，点击绿色按钮 **"Commit new file"**（提交新文件）

### 1.2 创建 images 文件夹

1. 再次点击 **"Create new file"** 按钮
2. 在文件名输入框输入：`images/.gitkeep`（这个文件只是用来占位置的）
3. 向下滚动，点击绿色按钮 **"Commit new file"**

好的！图片仓库配置完成了！✅

---

## 🔑 第二步：生成 GitHub 访问令牌

### 2.1 打开设置页面

1. 点击 GitHub 右上角你的头像
2. 在弹出的菜单中点击 **"Settings"**（设置）

### 2.2 找到开发者设置

1. 在设置页面左侧菜单，一直向下滚动，找到最底部的 **"Developer settings"**（开发者设置）
2. 点击它

### 2.3 生成令牌

1. 在左侧菜单点击 **"Personal access tokens"** → 选择 **"Tokens (classic)"**
2. 点击右上角的 **"Generate new token"** → 选择 **"Generate new token (classic)"**
3. 这时候可能需要你输入 GitHub 密码确认

### 2.4 填写令牌信息

现在你看到了一个表单，按下面填写：

- **Note**（说明）：输入 `Photo Gallery Upload`
- **Expiration**（有效期）：选择 `No expiration`（永久有效，省事）
- **Select scopes**（选择权限）：找到 `repo`，勾选前面的复选框（勾选后 repo 下面的子选项会自动全选）

### 2.5 保存令牌！

1. 向下滚动到最底部，点击绿色按钮 **"Generate token"**
2. **重要！！！** 现在你会看到一串字母和数字（像 `ghp_xxxx...`）
3. **立即复制这串字符，保存在记事本里！** 关闭页面后就再也看不到了！

好的！令牌生成完成！✅

---

## 🌐 第三步：配置 Vercel

### 3.1 打开 Vercel 项目

1. 访问 https://vercel.com/dashboard
2. 找到你的项目，点击进入

### 3.2 进入设置页面

1. 在项目页面顶部，找到并点击 **"Settings"**（设置）标签页
2. 在左侧菜单找到 **"Environment Variables"**（环境变量）

### 3.3 添加第一个环境变量

1. 在 **Name**（名称）输入框输入：`GITHUB_TOKEN`
2. 在 **Value**（值）输入框粘贴你刚才复制的那串字符（ghp_开头的）
3. 点击右边的 **"Save"**（保存）按钮

### 3.4 添加第二个环境变量

1. 再次点击 **"Add new"**（添加新的）
2. **Name** 输入：`REACT_APP_GITHUB_OWNER`
3. **Value** 输入：你的 GitHub 用户名（不是邮箱！就是你 GitHub 头像旁边显示的名字）
4. 点击 **"Save"**

### 3.5 添加第三个环境变量

1. 再次点击 **"Add new"**
2. **Name** 输入：`REACT_APP_GITHUB_REPO`
3. **Value** 输入：`photo-gallery-storage`
4. 点击 **"Save"**

好的！环境变量配置完成！✅

---

## 🚀 第四步：部署更新

### 4.1 重新部署项目

1. 在 Vercel 项目页面，点击顶部的 **"Deployments"**（部署）标签页
2. 找到最新的一次部署（最上面那个）
3. 点击右边的三个点（更多选项）
4. 在弹出的菜单中点击 **"Redeploy"**（重新部署）
5. 等待部署完成（大概需要1-2分钟）

好的！部署完成！✅

---

## ✅ 第五步：提交我们刚才修改的代码

### 5.1 提交代码到 GitHub

现在我们需要把刚才优化的代码（使用了 CDN 加速）推送到 GitHub：

```bash
# 在你的项目文件夹里，打开终端/命令行，输入：
git add .
git commit -m "优化：使用 jsDelivr CDN 加速国内访问"
git push
```

（如果你用的是 GitHub Desktop，直接点提交然后推送就可以了）

### 5.2 等待 Vercel 自动部署

推送代码后，Vercel 会自动检测到并重新部署，不用你手动操作！

## 🎉 第六步：测试功能

### 6.1 打开你的网站

部署完成后，访问你的 Vercel 网站地址

3. 应该能看到刚才上传的图片了！

## ❓ 常见问题

### Q1: 找不到某个页面/按钮怎么办？

**A:** 别着急，GitHub 和 Vercel 偶尔会更新界面。找不到的时候看看页面上的提示文字，或者搜索一下。

### Q2: 上传图片报错怎么办？

**A:** 
1. 检查三个环境变量是否都正确添加了
2. 确认 GitHub token 还没过期
3. 查看 Vercel 项目的部署日志（在 Deployment 页面可以看到）

### Q3: 图片加载很慢怎么办？

**A:** 别担心，我们已经配置了 jsDelivr CDN 加速！如果还是慢，可能是你的网络问题。

### Q4: 我想换一个图片仓库怎么办？

**A:** 
1. 在 GitHub 创建新仓库
2. 在 Vercel 环境变量里把 `REACT_APP_GITHUB_REPO` 改成新仓库名
3. 重新部署 Vercel
